# Test Objectives
---
* **Evaluating work products:** Reviewing requirements, user stories, designs , and code (***Static testing***).
* **Triggering failures:** Exwcuting the software to find defects (***Dynamic testing***).
* **Ensuring required coverage** of a test object (code, requirements, or risks).
* **Risk Reducing:**Proactively lowering the level of risk regarding inadequate software quality.
* **Verification:** Checking whether specified requirements have been fulfilled.
* **Compliance:** Verifying that a test object complies with contractual, legal, and requirements.
* **Stakeholder Support:** Providing necessary information to stakeholders to allow them to make informed decisions.
* **Confidence Building:** Creating trust in the quality level of the test object.
* **Validation:** Checking whether the test object is complete and works as expected by the stakeholders.

---
## Objectives of testing can vary, depending on: 
* what work producted being tested
* the test level
* risks
* SDLC being followed
* corporate structure
* competitive considerations
* time to market
